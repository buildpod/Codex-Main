# Anti-Hallucination Rules

Status: Draft baseline for Factory 1.0

## Purpose

These rules prevent Factory work from being driven by unsupported claims, tool hype, or invented requirements.

## Rules

1. No tool recommendation without purpose, risk, alternative, and decision.
2. No product requirement without approved product context or explicit assumption label.
3. No compliance, security, or legal claim without source and human approval.
4. No code acceptance without tests or documented reason tests could not run.
5. No release readiness claim without evidence records.
6. No model/tool output is trusted as final evidence by itself.
7. Unsupported statements must be labeled as assumptions.
