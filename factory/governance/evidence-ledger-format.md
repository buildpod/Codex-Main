# Evidence Ledger Format

Status: Draft baseline for Factory 1.0

## Purpose

Define the structure of evidence records used by Factory.

## Evidence Record Fields

- Evidence ID:
- Date/time:
- Product:
- Ticket/decision/release ID:
- Agent/tool:
- Human owner:
- Source documents:
- Files changed:
- Commands run:
- Test/scan results:
- Screenshots/artifacts:
- Risks:
- Assumptions:
- Approval status:
- Next action:

## Evidence Types

- Intake evidence.
- Ticket evidence.
- Test evidence.
- Security evidence.
- Architecture evidence.
- Decision challenge evidence.
- Release evidence.

## Rule

Evidence can be Markdown for Factory 1.0. Hash chaining is optional later.
