# Incident Response Lite

Status: Draft baseline for Factory 1.0

## Purpose

Define first actions when something goes wrong.

## Incidents

- Secret leaked.
- Suspicious dependency installed.
- AI exposed private data.
- Repo corrupted.
- Security scan finds critical issue.

## First Response

1. Stop affected work.
2. Preserve evidence.
3. Rotate exposed secrets.
4. Identify affected files, repos, and tools.
5. Record corrective action.
6. Update Factory rules if needed.
